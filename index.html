Ajax

AJAX(Asynchronous Javascript And XML)

在AJAX技术出现之前,如果想局部的更新页面信息,必须要刷新整个页面.递交表单,查询信息…这些操作在那个时代的体验比较糟糕.
AJAX技术的出现解决了这个问题.通过与服务器进行少量数据交互,AJAX实现了异步更新.

具体来说,AJAX包括以下步骤

  1. 创建AJAX对象
  2. 发出HTTP请求
  3. 接手服务器传回的数据
  4. 更新网页数据

XMLHttpRequest

XMLHttpRequest对象用于在浏览器和服务器之间传递数据.

  • readyState
    readyState是一个表示XMLHttpRequest请求当前所处状态的只读属性
    通信过程中,每当状态发生一次变化,readyState属性的值就会发生改变.readyState的值没变化一次,都会触发readyStateChange事件
    • 0: 请求未初始化
    • 1: 服务器连接已建立
    • 2: 请求已接收
    • 3: 请求处理中
    • 4: 请求已完成，且响应已就绪

  • onreadystatechang
    onreadystatechang属性指向一个回调函数,当readyStateChange事件发生时,所指向的回调函数就会被调用,与此同时readyState属性也会发生改变.
    此外,使用abort()方法,终止XMLHttpRequest请求,onreadystatechang指向的回调函数也会被调用

  • response
    response是一个用于返回接收到的数据的只读属性.
    它的类型可以是ArrayBuffer,Blob,Document,JSON或者是一个字符串.
    其类型由XMLHttpRequest.responseType的值决定
    如果本次请求失败或者数据不完整,该属性值为null.

  • responseType
    responseType属性用来指定服务器返回类型
    • "": 字符串(默认值)
    • arraybuffer: ArrayBuffer对象
    • blob: Blob对象.适合读取二进制数据,比如图片文件
    • document: Documentd对象
    • json: JSON对象
    • text: 字符串.适合大多数情况,直接处理文本比较方便
  • responseText
    responseText是一个返回从服务器接受的字符串的只读属性
    如果本次请求失败或者数据不完整,该属性就为null
    如果服务器返回的数据格式是JSON就可以使用responseText属性
  • responseXML

正则表达式

在处理字符串的时,常会有查找符合某些特定规则的字符串的需求.
正则表达式就是用于描述这些规则的工具.

JavaScript的正则体系是参照Perl语言建立的.

建立正则表达式有两种方法。一种是使用字面量，以斜杠开始,以斜杠结束。

另一种是使用RegExp 构造函数(用new创建出来的东西都是对象)

这两种写法都新建了一个内容为poi的正则表达式对象

第二种写法在运行时新建正则表达式,正则对象在代码运行到这里时生成

字符串对象的方法

  • String.prototype.match()
    match() 方法用于检索字符串内指定的值，或通过正则进行匹配.
    返回匹配结果(以数组形式),匹配失败返回null
    参数为指定的值或者正则表达式

    如果正则表达式带有g修饰符，则该match方法与正则对象的exec方法行为不同，会一次性返回所有匹配成功的结果。
    正则表达式的lastIndex属性，对match方法无效，匹配总是从字符串的第一个字符开始。

  • String.prototype.search()
    search() 方法用于检索字符串中指定的子字符串，或检索与正则表达式相匹配的子字符串。
    返回第一个与指定的子字符串或者与 正则表达式 相匹配的子串的起始位置。
    没有找到则返回 -1
    参数为指定的子字符串 或者正则表达式

    search() 方法不执行全局匹配，它将忽略标志 g。
    它同时忽略 正则 的 lastIndex 属性，并且总是从字符串的开始进行检索
    这意味着它总是返回 字符串 的第一个匹配的位置

  • String.prototype.replace()
    replace方法可以替换匹配到的值。
    它接受两个参数，第一个是搜索规则(指定的值或者正则)
    第二个是替换的内容。也可以是
    返回修改后的字符串.匹配失败,即没有修改则返回原字符串.

    无论匹配是否成功,原字符串不发生改变!

    replace 方法的第二个参数可以使用$，用来指代所替换的内容。

    • $& 指代匹配的子字符串
    • $` 指代匹配结果前面的文本
    • $' 指代匹配结果后面的文本
    • $n 指代匹配成功的第n组内容，n是从1开始的自然数。
    • $$ 指代美元符号$

    对于$n的一些用法仍有不明白的,此处应在未来补全!

    replace方法的第二个参数还可以是一个函数，将每一个匹配内容替换为函数返回值。
    对于这个函数的参数问题存疑,需要在未来解决!

  • String.prototype.split()
    split方法按照规则分割字符串，返回一个由分割后的各个部分组成的数组。
    该方法接受两个参数，第一个参数是分隔规则，第二个参数是返回数组的最大成员数(可选)。

匹配规则(重点)

文本字符

如果某个字符只表示它字面的含义,那么它们就叫做文本字符(字面量字符)

元字符

还有一部分字符不代表字面意思,这类被称为元字符
要匹配这些字符本身时需要

  • \ 将下一个字符标记为一个特殊字符、或一个原义字符、或一个向后引用(后顾)、或一个八进制转义符
    例如，’n’ 匹配字符 “n”。’\n’ 匹配一个换行符
    ‘\‘ 匹配 “\” 而 “(“ 则匹配 “(“

  • .
    匹配除回车\r、换行\n 、行分隔符\u2028和段分隔符\u2029以外的所有字符

    这个正则中会匹配a和b之间包含任意一个字符的情况.
    aAb,aPb,a1b都会被匹配到,而ab,a22b则不会被匹配到.

  • ^表示字符串的开始位置,如果开启多行模式，^ 也匹配 ‘\n’ 或 ‘\r’ 之后的位置。

  • $表示字符串的结束位置,如果开启多行模式$ 也匹配 ‘\n’ 或 ‘\r’ 之前的位置。

  • x|y 表示匹配x 或y
    /t|bag/表示匹配t或者bag
    /(t|b)ag/表示匹配tag或bag

其他元字符还包括 \\、\*、+、? 、()、[]、{}等,将在下文解释。

    末尾添加一个或多个元素，并返回新的length

  • shift()

    删除第一个元素,返回删除的元素的值(若已为空，则不改变数组,返回undefined)

  • unshift()

    数组的开头添加一个或多个元素，并返回新的length

  • splice()

    添加/删除元素，然后返回被删除的元素

    • start​
      从数组的哪一位开始修改内容。如果超出了数组的长度，则从数组末尾开始添加内容;
      如果是负值，则表示从数组末位开始向左计数

    • deleteCount
      表示要移除的数组元素的个数,为整数
      如果 deleteCount 是 0，则不移除元素。这种情况下，至少应添加一个新元素.
      如果 deleteCount 大于start 之后的元素的总数，则从 start 后面的元素都将被删除（含第 start 位）。

    • itemN
      要添加进数组的元素。如果不指定，则 splice() 只删除数组元素。

    • 返回值
      由被删除的元素组成的一个数组。如果只删除了一个元素，则返回只包含一个元素的数组。如果没有删除元素，则返回空数组。

    以上五种方法都会对原数组进行修改

  • slice()

    从已有的数组中选取元素存入到新数组中,返回新数组(不会改变原数组)

    从star开始(包括star)在end结束(不包括end)

转换

  • join()

    把所有元素放入一个字符串,通过传入的参数连接各元素,不改变原数组
    如果传入的参数是空字符串,那么数组中的所有元素将被直接连接。
    如果省略传入的参数，默认用逗号连接

  • toString()

    把数组转换为字符串，并返回结果,不改变原数组

  • split
    把一个字符串分割成字符串数组

    • separator
      必需,字符串或正则表达式，从该参数指定的地方分割stringObject.
      如果把空字符串 (“”) 用作 separator，那么 stringObject 中的每个字符之间都会被分割。
    • howmany
      可选,该参数指定返回的数组的最大长度。如果设置了该参数，返回的子串不会多于这个参数指定的数组。
      如果没有设置该参数，整个字符串都会被分割，不考虑它的长度。
    • 返回值
      返回一个由字符串元素组成的数组。
      该数组是通过在 separator 指定的边界处将字符串 stringObject 分割成子串创建的。
      返回的数组中的字串不包括 separator 自身
      但是，如果 separator 是包含子表达式的正则表达式，那么返回的数组中包括与这些子表达式匹配的字串（但不包括与整个正则表达式匹配的文本）

String.split() 执行的操作与 Array.join 执行的操作是相反的

排序

    方法用于颠倒数组中元素的顺序

  • sort()
    方法用于对数组的元素进行排序

  • compareFunction

      可选.用来指定按某种顺序进行排列的函数。
如果省略，元素按照转换为的字符串的诸个字符的Unicode位点进行排序

    • 记 a 和 b 是两个将要被比较的元素：

      • 如果 compareFunction(a, b) 大于 0 ， b 会被排列到 a 之前。
      • compareFunction(a, b) 必须总是对相同的输入返回相同的比较结果，否则排序的结果将是不确定的。

Date

Date

Date 对象用于处理日期和时间.

Date 对象基于1970年1月1日起的毫秒数开始计数.

Date()

Date() 对象可以作为普通函数直接调用，返回一个代表当前时间的字符串

即使带有参数，Date()作为普通函数使用时，返回的还是当前时间。

new Date()

只能使用 new Date()构造时间对象
注意new Date()和Date()区别

  • 参数为空时
    返回当前时间
  • value(只有一个数值参数时)
    返回自1970年1月1日00:00:00 (世界标准时间) 起经过的毫秒数。
  • dateString
    表示日期的字符串。参数为一个能被 Date.parse()方法识别的字符串
  • 参数为两个及两个以上时
    • year
      代表年份的整数值。为了避免2000年问题最好指定4位数的年份; 使用 1998, 而不要用 98.
    • month
      代表月份的整数值从0（1月）到11（12月）
    • day
      代表一个月中的第几天的整数值，从1开始。
    • hour
      代表一天中的小时数的整数值 (24小时制)。
    • minute
      分钟数
    • second
      秒数
    • millisecond
      毫秒数

注意 :不加 new 操作符将返回的只是一个字符串，而不是一个日期对象

构造时间对象时，如果传入的参数大于合理范围时,数值会被做出相应的调整。

例如

如果没有输入任何参数，则会依据系统当前时间来创建一个Date对象
如果提供了至少两个参数，其余的参数均会默认设置为1或者0。

Date 对象方法

  • getDate() 从 Date 对象返回一个月中的某一天 (1 ~ 31)

  • getDay() 从 Date 对象返回一周中的某一天 (0 ~ 6)

  • getMonth() 从 Date 对象返回月份 (0 ~ 11)

  • getFullYear() 从 Date 对象以四位数字返回年份

  • getYear() 请使用 getFullYear() 方法代替

  • getHours() 返回 Date 对象的小时 (0 ~ 23)

  • getMinutes() 返回 Date 对象的分钟 (0 ~ 59)

  • getSeconds() 返回 Date 对象的秒数 (0 ~ 59)

  • getMilliseconds() 返回 Date 对象的毫秒(0 ~ 999)

  • getTime()
    返回从1970-1-1 00:00:00 UTC（协调世界时）到该日期经过的毫秒数，对于1970-1-1 00:00:00 UTC之前的时间返回负值。
