数据类型转换

JavaScript 是一种动态类型语言,变量没有类型限制,可以随时赋予任意值.

虽然JS中变量没有类型，但是数据本身和各种运算符是有类型的。
如果运算符发现，数据的类型与预期不符，就会自动转换类型.
比如，减法运算符预期两侧的运算子应该是数值，如果不是，就会自动将它们转为数值。

'4' - '3'//1

JS中类型转换是个令人头疼的问题.
本篇讲述强制转换和自动转换的规则.

强制转换

强制转换主要指使用Number、String和Boolean三个构造函数，手动将各种类型的值，转换成数字、字符串或者布尔值

Flex

Flex之前用什么?

  • normal flow
  • float + clear
  • position relative + absolute
  • display inblock-inblock
  • 负 margin

Flex 来了

块级元素垂直排列,行内元素水平排列.
flex 则与方向无关.
flex 空间自动分配,自动对齐.
flex 适用于简单的线性布局,更复杂的布局交给 grid 布局(请期待下个版本)

flex不翻译为弹性布局,因为之前通过百分比的方式的布局已经被成为弹性布局.

Flex 的属性

Flex container 的属性

  • flex-direction 方向
    • row(默认): 主轴为水平方向,起点在左端.
    • row-reverse: 主轴为水平方向,起点在右端.
    • column: 主轴为垂直方向,起点在上沿.
    • column-reverse: 主轴为垂直方向,起点在下沿.
      demo
  • flex-wrap 换行
    • nowrap(默认值):不换行.
    • wrap: 换行,第一行在上方.
    • wrap-reverse: 换行,第一行在下.
      demo
  • flex-flow 上面两个元素的简写
    flex-flow: <flex-direction> || <flex-wrap>;
  • justify-content 主轴方向对齐方式
    • flex-start(默认值): 左对齐.
    • flex-end: 右对齐.
    • center: 居中.
    • space-between: 两端对齐,item之间的间隔都相等
    • space-around: 每个item两侧的间隔相等.所以,项目之间的间隔比项目与边框的间隔大一倍
      demo
  • align-items 侧轴方向对齐方式
    • flex-start: 侧轴起点对齐.
    • flex-end: 侧轴终点对齐.
    • center: 侧轴中点对齐.
    • baseline: item的第一行文字的基线对齐.
    • stretch: 如果item未设置高度或设为auto,将占满整个容器的高度.
      demo
  • align-content 多行/列内容对齐方式
    • 极少用到.

Flex item 的属性

  • flex-grow (剩余空间多时,剩余空间如何分配)定义项目的放大比例.默认为0,值越大分配到的空间越多.
  • flex-shrink(空间不够时,item如何瘦身)定义项目的放大比例,默认为0.值越大,瘦身程度越大.
  • flex-basis(一般不用) 默认占用大小.不写的话就是原始大小
  • flex 上面三个的缩写. flex: none | [ <'flex-grow'> <'flex-shrink'>? || <'flex-basis'> ]. 建议优先使用这个属性.
  • order 顺序 有了这个就可以不用双飞翼了.通过CSS来控制顺序.
  • align-self 允许单个item有与其他item不一样的对齐方式,可覆盖align-items属性.默认为auto,表示继承父元素的align-items属性.如果没有父元素,则等同于stretch.

模块化

为什么要使用模块化？

首先说说为什么需要模块化

小明和小王各自负责实现不同的两个功能.
两个人分开写的代码最终会在一个应用同时上使用.
两个人事先没做什么约定.结果

两个人的代码因为命名冲突,部分全局变量相互覆盖
两个模块都用到了一些相同的功能,但是两人都各自完成,浪费时间.
最终这个应用炸了…..
两人加班到深夜,不断调试,最终才在Deadline前勉强上线.
更惨的是后来的小李,他要负责维护两人离职后留下的代码……

模块化的意义

  • 解决命名冲突.
  • 减少代码耦合性,便于依赖管理.
  • 提高代码复用性,减少开发成本.
    比如模块1,模块2都需要同一个方法来实现,那么就把这个方法抽离出来做成模块3.
    在1,2中调用模块3.
  • 便于使用,想用哪个功能就加载哪个模块.迭代升级添加新功能也较为方便.
  • 方便阅读和维护.

简单的模块化写法

最原始的写法

这种写法中一个函数就是一个模块,使用哪个模块直接调用相应函数.
缺点很明显,看不出各个模块之间有什么关系
一个模块占据一个全局变量名称,变量名冲突依旧可能发生

对象写法

把上面的那种写法封装进一个对象里,这样全局变量的问题就解决了.

但是问题依旧存在,模块内部的成员全部暴露在外.
万一别的代码中在全局环境执行了一句block1.cur = "Boom!".(也可能是第三方注入)
模块内的重要参数被改变了,模块功能还要出错.

立即执行函数写法

对上面的写法再做点改动.把模块放进一个立即执行函数,这样cur就无法再外部读取了.

模块化的规范

使用模块化的写法有个前提,就是所有的开发人员遵循同一套规则.

，那就是大家必须以同样的方式编写模块，否则你有你的写法，我有我的写法，岂不是乱了套！考虑到Javascript模块现在还没有官方规范，这一点就更重要了。

transform 2D

平面旋转

rotate()是元素相对圆点中心旋转,值为正顺时针,值为负就是逆时针.

####　扭曲
skew()让元素倾斜显示.可以让元素以其中心位置围绕X轴或Y轴倾斜
rotate只是旋转,而skew只是改变形状.
用法
skew(x,y),skewX(),skewY()

缩放

沿着X Y方向缩放
scale(x,y),scaleX(),scaleY()
值为缩放倍数.

位移

translate.沿着XY轴移动,不影响其他组件

####　变换矩阵
matrix(a,b,c,d,e,f)
上边四种效果都可以使用矩阵来表示,实际上上述四种效果都是依托于matrix来实现的
(不就是线性代数哪点玩么~)
a:控制缩放
d:控制缩放
e:水平偏移距离
f:垂直偏移距离
除此之外还有matrix3D()用于3D变换

####　设置原点
默认是 XY50%的地方
transform-origin

####　背景图大小

1、auto：默认值，不改变背景图片的原始高度和宽度；

2、<长度值>：成对出现如200px 50px，将背景图片宽高依次设置为前面两个值，当设置一个值时，将其作为图片宽度值来等比缩放；

3、<百分比>：0％~100％之间的任何值，将背景图片宽高依次设置为所在元素宽高乘以前面百分比得出的数值，当设置一个值时同上；

4、cover：顾名思义为覆盖，即将背景图片等比缩放以填满整个容器；

5、contain：容纳，即将背景图片等比缩放至某一边紧贴容器边缘为止。

多重背景

如果有 size 值，需要紧跟 position 并且用 “/“ 隔开；
如果有多个背景图片，而其他属性只有一个（例如 background-repeat 只有一个），表明所有背景图片应用该属性值。
background-color 只能设置一个。
