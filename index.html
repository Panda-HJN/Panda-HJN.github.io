实现一个日历插件

这篇博客的目标是要完成这样一个demo.

明确需求

  1. 在一页日历中,包含本月的所有日期.用黑色字体显示.被选中的日期用红色加粗字体表示.
    包含上个月的最后几天和下个月的前几天.用灰色字体表示.不可被选中.
  2. 点击前进,后退两个按钮日历将会跳转到前一月/下一月.
  3. 当鼠标悬浮在某个日期时,这个日期会出现一个灰色的矩形.
  4. 日历根据所选择的日期,会返回这个日期的时间对象.并在框体内以YYYY/MM/DD的形式展示.
  5. 每个日历组件创建后默认选择当前日期,或者由HTML元素中的属性来决定当前选中的日期
  6. 日历组件可在页面的重复使用

HTML

HTML结构如下.

页面的源文件中只需要有一个如下的input标签.
date-init 属性设置初始时间,如果缺省则选当前时间.
这个input标签的jQuery元素将会作为参数传入到日历的构造函数中.

整个日历的基本模板由插件内的 render方法 动态生成.
每个月具体的日期存放在<tbody>内.由插件的setDate方法动态生成并排版.
下面是个范例.

在 tbody 中由JavaScript来动态生成日历.
每行一组 <tr></tr> 内有七个 <td></td> .七天一个星期
上个月的 <td> 的 class 为 pre-month.
下个月的 <td> 的 class 为 next-month.
本月的 <td> 的 class 为 cur-month.
被选中的 <td> 的 class 额外添加 cur-date.

JS部分

准备工具函数

通过分析,发现正确的排版日历需要有如下几个功能来支持

  1. 传入一个 date ,返回 date 所在月第一天的时间对象
  2. 传入一个 date ,返回 date 所在月最后一天的时间对象
  3. 传入一个 date ,返回上个月第一天的时间对象
  4. 传入一个 date ,返回下个月第一天的时间对象
  5. 判断input元素设置的表示初始时间的字符串是否合法
  6. 把时间以YYYY/MM/DD的形式返回.把1改写为01

下面就写出函数来实现这几个功能.

getFirstDay

getFirstDay方法可以获取 date 所在月份第一天的时间对象.
这个方法实现起来很简单.
只需要把 date 的 年 月 和 1 做为参数来生成一个时间对象即可

getLastDay

getLastDay方法可以获取 date 所在月份最后一天的时间对象.
一个月可能有30,31,28,29天.但是每天都有1000 * 60 * 60 * 24毫秒.
可以先获得下个月的第一天的时间戳,再减去1000 * 60 * 60 * 24.
这样就能得到date所在月份的最后一天的时间戳了.

需要注意到问题是12月和1月交替时的情况.

getPreMonth

获取 date 上个月 第一天的时间对象

getNextMonth

获取 date 下个月第一天的时间对象

isValidDate

判断字符串是否能被 Date() 识别

getYYMMDD

把时间以yyyy/mm/dd的形式返回出来.
同时要把个位数字前加0.

如何摆放tbody里的内容?

由已经完成的几个工具函数可以得到所有应该显示在日历内的日期的所有时间对象.
把所有的时间对象放入一个空数组内.并且标注好是上个月的下个月的还是这个月的日期.
然后由通过遍历这个数组来拼装 tbody里的内容.

call()与this

this 是一个代表当前执行上下文的隐藏参数.
this只有在执行的时候才能知道是什么.
所以this在不同场景代表的是什么是个有点令人头疼的问题.
在说明this之前先回顾下函数的调用.
函数调用有这几种方式

前两种方式其实相当于第三种方式的语法糖.

其中call()的第一个参数用于指定当前的执行上下文,也就是this.
后面的参数是函数执行时传入的参数.
这么来看this就好理解了.

apply()的与call()类似,只不过apply()只有两个参数.
第一个指定执行上下文,第二个是一个储存所有参数的数组.

三条规则

然后只要使用下面三点规则就很好判断.

  1. this 就是 call 一个函数时，传入的执行上下文环境.是call的第一个参数.
  2. 如果函数调用形式不是 call 形式，请转换为 call 形式来分析
  3. 全局函数是window的方法.所以func()等价于window.func()等价于window.func.call(window).

定时器和事件绑定

  • 事件的回调函数中的 this 是触发事件dom节点

  • setTimeout,setInterval
    这两个方法的执行上下文与调用他们的函数的执行上下文是分离的.
    这两个方法都是在 window 下执行的.
    表现有:无论这两个方法在哪执行,返回的的整数值都是连续的.
    所以这两个方法的 this 都是 window.

跨域

同源政策

一个完整的URL如
http://user:pass@www.hellopanda.com:80/list/index.html?uid=111#ppp
由以下部分组成

  • 协议:http,https,ssh,file,ftp,mailto…..
  • 登录信息:不常见,有时候SVN会用到
  • 域名: www.hellopanda.com
  • 端口: http默认80,https默认443,ftp默认21
  • 资源位置: 描述资源在服务器所处的位置
  • 查询字段: 向服务器递交字段从而获得相应信息

  • 片断标识符hash: 根据锚点用于文档内的定位,浏览器内的行为,不会发送到服务器

    同源政策的指的是同协议,同域名,同端口.只要有一个不相同,就不符合同源政策
    例如
    http://a.com/index.php和http://a.com/main.js 同源
    http://a.com/index.php和https://a.com/main.js协议不同,不同源
    http://a.com/index.php和http://bbs,a.com/main.js域名不同,不同源
    http://a.com:80/index.php和http://a.com:8080/main.js端口不同,不同源
    　如果不能满足同源的要求,会有如下的限制

    1. 不能使用AJAX获取资源
    2. 无法对Cookie、LocalStorage 和 IndexedDB 进行读取.
    3. DOM 无法获得

    从这些限制可以看出,同源策略的目的在于保证用户的信息安全.

    跨域发出 AJAX 请求在 network 里仍然可以看到 response .
    所以同源政策是通过浏览器阻止得到 response后的行为来实现的

跨域

同源政策带来了安全保障,但是也带来了限制
跨域就是突破同源政策的手段
跨域的方法有如下几种

JSONP

JSONP 是 JSON With Padding 的缩写.把JSON放到一个容器里的意思
JSONP 和 JSON 没有什么关系

原理

虽然无法向不同域发送 AJAX 请求来获得信息,但是浏览器并不阻止引用外域的静态资源,否则各种常用库的 CDN 还怎么用呢

JSONP 的原理就是利用这一特点去动态的加载script,后台根据src中相应的字段动态的给出 js文件.
把想要获得的信息放在js文件中方法的参数中

使用JSONP

首先在发起请求的页面声明一个函数

并根据函数名动态生成一个 scrpit 标签

这会引入otherside.com的一个js文件(后端根据callback=fun动态生成)
这个文件内会调用fun这个方法

其中的参数就是要跨域获取的信息

去调用哪一个方法 由callback=指定
其中callback以及函数名fun的名称都是由后端确定的,取其他的名字也可以,但是必须前后端保持一致

特点

  1. JSONP 的使用需要前后端同时配合才能实现
  2. 使用 JSONP 会增加全局变量,这一点需要注意
  3. 请求的地址必须是安全可信的站点,不然可能会在返回的方法中含有恶意代码,XSS攻击也常用JSONP.应对手段有使用正则对返回的内容进行检查
  4. 只能使用get方式

    CORS

    CORS 跨域资源共享(Cross-Origin Resource Sharing).是一种 AJAX 跨域请求资源方式
    当使用 XMLHttpRequest 发送请求时,浏览器发现该请求不符合同源策略,会给该请求加一个请求头:Origin
    后端接收到后作出相应处理,如果确定接受请求则在返回结果中加入一个响应头:Access-Control-Allow-Origin;
    浏览器判断该相应头中是否包含 Origin 的值,如果有则浏览器会处理响应,这样就可以拿到响应数据,否则浏览器直接驳回,这样就无法拿到响应数据.
    所以 CORS 的表象是让你觉得它与同源的 ajax 请求没啥区别,代码完全一样.

####　特点

  • 只支持IE10 以上　　　
  • 实现CORS的关键是服务器
  • 支持 HEAD,GET,POST三种方式

    降域

    当存在iframe嵌套,且一级域名相同时可使用此方法.
    让二者的window.domain值相同
    例1.a.com向域名为2.a.com的iframe跨域请求资源,只要将二者的window.domain都设置为a.com即可
  • 特点
  1. 仅适用于iframe嵌套的场景.应用场景有限
  2. 存在安全问题,如果子域名被攻击或者注入恶意代码,父域名将面临风险.

window.name

一个页面在其生命周期(在地址栏输入新的URL并连接意味着上一个生命周期结束)内 window.name的值不会发生改变.window.name的值永远是字符串

载入的iframe可通过window.name的值来传递信息从而实现跨域.

片断标识符

片段标识符(fragment identifier)指的是,URL的#号后面的部分
如果只是改变片段标识符,页面不会重新刷新.但是会触发hashchange事件,并在history中留下记录

  • 父窗口可以把信息,写入子窗口的片段标识符.子窗口通过监听 hashchange事件得到通知

  • 子窗口也可以改变父窗口的片段标识符

postMessage

HTML5标准推荐的跨域方案
这个方法也需要iframe

  1. 在a.com/a.html中插入载入b.com/b.html的 iframe
  2. 通过b.com/b.html对 b.com 发起postMessage请求
  3. 在b.com/b.html中添加message 监听事件,监听到请求之后验证发起请求的域名event.origin
  4. 然后通过ajax 获取数据，然后再创建一个postMessage 请求，此时设置为可被所有域名访问
  5. 在a.com/a.html中添加message 监听事件,获取并展示数据。

AJAX(Asynchronous Javascript And XML)

在AJAX技术出现之前,如果想局部的更新页面信息,必须要刷新整个页面.递交表单,查询信息…这些操作在那个时代的体验比较糟糕.
AJAX技术的出现解决了这个问题.通过与服务器进行少量数据交互,AJAX实现了异步更新.

具体来说,AJAX包括以下步骤

  1. 创建AJAX对象
  2. 发出HTTP请求
  3. 接手服务器传回的数据
  4. 更新网页数据

XMLHttpRequest

XMLHttpRequest对象用于在浏览器和服务器之间传递数据.

  • readyState
    readyState是一个表示XMLHttpRequest请求当前所处状态的只读属性
    通信过程中,每当状态发生一次变化,readyState属性的值就会发生改变.readyState的值没变化一次,都会触发readyStateChange事件
    • 0: 请求未初始化
    • 1: 服务器连接已建立
    • 2: 请求已接收
    • 3: 请求处理中
    • 4: 请求已完成，且响应已就绪

  • onreadystatechang
    onreadystatechang属性指向一个回调函数,当readyStateChange事件发生时,所指向的回调函数就会被调用,与此同时readyState属性也会发生改变.
    此外,使用abort()方法,终止XMLHttpRequest请求,onreadystatechang指向的回调函数也会被调用

  • response
    response是一个用于返回接收到的数据的只读属性.
    它的类型可以是ArrayBuffer,Blob,Document,JSON或者是一个字符串.
    其类型由XMLHttpRequest.responseType的值决定
    如果本次请求失败或者数据不完整,该属性值为null.

  • responseType
    responseType属性用来指定服务器返回类型
    • "": 字符串(默认值)
    • arraybuffer: ArrayBuffer对象
    • blob: Blob对象.适合读取二进制数据,比如图片文件
    • document: Documentd对象
    • json: JSON对象
    • text: 字符串.适合大多数情况,直接处理文本比较方便
  • responseText
    responseText是一个返回从服务器接受的字符串的只读属性
    如果本次请求失败或者数据不完整,该属性就为null
    如果服务器返回的数据格式是JSON就可以使用responseText属性
  • responseXML
